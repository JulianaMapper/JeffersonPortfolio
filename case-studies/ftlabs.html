The Problem

As CIO at Lake County Health Department, I spent four years experiencing firsthand what was broken in public health technology infrastructure. Local health departments were running on systems that couldn't talk to each other, couldn't scale, and weren't built by anyone who had actually run a health department.

When I went looking for a platform that could modernize public health data infrastructure — something cloud-native, AI-ready, HIPAA-compliant, and actually usable by under-resourced local departments — I couldn't find it. Enterprise health IT vendors build for hospital systems. Public health vendors build legacy software that looks like it hasn't changed since 2003. Nobody had built the thing the market actually needed.

The Decision

I co-founded F&T Labs to build the platform I couldn't find when I was a buyer. The decision to build on AWS from the ground up — rather than layering onto an existing SaaS product — was deliberate. I'd been the CIO who got burned by that approach. Platforms that patch HIPAA compliance onto an existing architecture are never actually compliant-by-design. They're compliant-by-policy, and policy erodes.

The design constraints were non-negotiable from day one: equity-centered, HIPAA-compliant as a structural property not a checklist item, multi-tenant so it could serve organizations of different sizes, and AI-ready at the infrastructure level — not bolted on after the fact.

What I Built

PH360 (Public Health 360) is a 7-layer, HIPAA-compliant, multi-tenant AI platform on AWS. I am the architect of record — I designed the architecture, wrote the CloudFormation templates, and built the Lambda functions. This is not a case where the CEO oversees an engineering team. I built it.

PH360 — 7-Layer AWS Architecture
1
Presentation Layer
User-facing interfaces for health department staff and administrators. Designed for non-technical users who can't afford a steep learning curve.
2
API Gateway
AWS API Gateway — secure, scalable request routing. Rate limiting and authentication enforcement at the gateway level.
3
Authentication & Authorization
AWS Cognito — multi-tenant identity management, role-based access control, MFA enforcement. Compliance-by-design, not by policy.
4
Compute Layer
AWS Lambda — serverless compute for data processing pipelines, automation workflows, and AI inference calls. Zero idle infrastructure cost.
5
AI / Knowledge Layer
AWS Bedrock + Claude LLM — RAG-based knowledge system for public health data analysis. Automation pipelines for reporting and surveillance workflows.
6
Data Storage
AWS S3 — encrypted at rest, versioned, lifecycle-managed. Structured for HIPAA-compliant PHI handling and audit trail requirements.
7
Infrastructure as Code
AWS CloudFormation — every resource defined in code. Reproducible, auditable, deployable. No snowflake infrastructure.

The Result

  • 30× year-over-year revenue growth — from $22K to $500K+ in 12 months through strategic business development and government RFP capture
  • PH360 in active production serving local health departments
  • WBE and SBA 8(a) certified — positioned for government contracting at scale
  • Built simultaneously with full founder-operator responsibilities across CEO, CFO, CRO, and CIO functions
